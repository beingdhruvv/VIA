# AGENT PROTOCOL

## Session START — do in order
1. Read HANDOFF STATE. Understand status, blockers, next step.
2. Run: git log -5 --oneline to confirm current commit.
3. Check FILE MAP for areas you will touch.
4. Never trust stale notes — always verify the actual file first.

## Session END — mandatory before closing
1. Rewrite HANDOFF STATE: date+time, summary, blockers, next step.
2. Update WORKING DIRECTORY if files were added/renamed/deleted.
3. Append a row to DECISION LOG for every non-obvious choice made.
4. Append a row to VALIDATION LOG if build ran or tests done.
5. Update CONTEXT FILE LOG at the bottom.

## What agents must never do
- Never commit secrets, API keys, or credentials.
- Never write AI/agent/IDE name in commit messages or code comments.
- Never mark build as passing without actually running: npm run build
- Never leave HANDOFF STATE blank or with a stale date.

## Deployment
- Production URL, droplet addresses, SSH: **`LOCAL_OPERATIONS.md`** only (copy from `LOCAL_OPERATIONS.md.template`; never commit the filled file).
- App on server: `/var/www/via`
- Deploy: push to main → GitHub Actions → `infra/server/deploy.sh`
- DB: PostgreSQL in production (`schema.production.prisma`)
- Auth: NextAuth v5 JWT + Firebase Google Sign-In

## Known issues / caveats
- Prefer a passphrase-free **deploy key** for CI; keep a personal passphrase key only on your machine.
- NextAuth serverActions.allowedOrigins experimental warning (non-breaking)
- Prisma enums stored as String in SQLite; switch to postgresql + real enums before prod deploy
- Firebase: set `NEXT_PUBLIC_FIREBASE_*` in env; add your production hostname under Firebase Console → Authentication → Authorized domains

# PROJECT SNAPSHOT

Purpose: VIA — personalized multi-city travel planning web app for Odoo Hackathon 2026 (Team StormLabs)

Stack:
- Frontend: Next.js 16.2.6 (App Router), TypeScript, Tailwind CSS v4
- Auth: NextAuth v5 beta (Credentials + JWT)
- DB (local): SQLite via Prisma 5
- DB (production): PostgreSQL via Prisma 5
- UI: Custom Tailwind components + Radix UI headless primitives
- Charts: Recharts
- Maps: react-leaflet + Leaflet
- Drag/Drop: dnd-kit
- Forms: React Hook Form + Zod
- State: Zustand
- Animations: Framer Motion
- Icons: Lucide React
- Hosting target: DigitalOcean Droplet (Ubuntu 24.04, Nginx + PM2)

# HARD RULES

## Never do
- Never use gradient backgrounds in UI
- Never use border-radius > 4px (except avatars = 50%)
- Never use emojis in UI
- Never use Bootstrap, MUI, Chakra, Ant Design
- Never use blurry/soft shadows — only 3px offset brutalist shadow
- Never hardcode city/activity data in components — always from DB
- Never write AI/agent/IDE name in commit messages
- Never commit .env.local or .env.production files
- Never mark build passing without running npm run build

## Always do
- All styling via Tailwind utility classes — no inline styles
- All forms use React Hook Form + Zod with inline error messages
- All numbers/prices/dates use font-mono (IBM Plex Mono)
- All cards use 1px solid #111111 border + 3px 3px 0px #111111 shadow
- All API routes return proper HTTP status codes
- Use Lucide React icons only — never custom SVGs for icons
